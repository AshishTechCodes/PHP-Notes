PHP Object-Oriented Programming Concepts

1. Class and Object

A class is a blueprint for creating objects. An object is an instance of a class. 

<?php
class Fruit {
    public $name;
    public $color;
    
    function set_name($name, $color) {
        $this->name = $name;
        $this->color = $color;
    }
    
    function get_name() {
        return "Fruit name is {$this->name} and color is {$this->color}.";
    }
}

$apple = new Fruit();
$apple->set_name('Apple', 'red');
echo $apple->get_name();
?>

Explanation: Here we create a Fruit class with properties and methods, then instantiate it as an object.

2. Constructor

__construct() is a special method called automatically when the object is created. 

<?php
class Car {
    public $name;
    public $color;
    
    function __construct($name, $color) {
        $this->name = $name;
        $this->color = $color;
    }
    
    function get_name() {
        return $this->name;
    }
    
    function get_color() {
        return $this->color;
    }
}

$bmw = new Car("BMW", "red");
echo $bmw->get_name();
echo "<br>";
echo $bmw->get_color();
?>

Use case: Cleaner and faster object initialization when you already know the values at creation time.

3. Destructor

__destruct() is called automatically when the object is destroyed or script ends. 

<?php
class Book {
    public $name;
    public $color;
    
    function __construct($name, $color) {
        $this->name = $name;
        $this->color = $color;
    }
    
    function __destruct() {
        echo "The book is {$this->name} and color is {$this->color}.";
    }
}

$physics = new Book("Java", "yellow");
?>

Use case: When you need to do something right before the object is destroyed, like saving state or freeing resources.

4. Inheritance

Inheritance allows a class to inherit properties and methods from another class.

Single Inheritance

<?php
class Pen {
    public $name;
    public $color;
    
    public function __construct($name, $color) {
        $this->name = $name;
        $this->color = $color;
    }
    
    public function intro() {
        echo "A {$this->name} is a Pen and color of pen is {$this->color}.";
    }
}

class NewPen extends Pen {
    public function message() {
        echo "New pen are not write that much good";
    }
}

$pen_new = new NewPen("Fountain", "red");
$pen_new->message();
echo "<br>";
$pen_new->intro();
?>

Multilevel Inheritance

<?php
class Limit {
    public $num1;
    public $num2;
    
    public function __construct($num1, $num2) {
        $this->num1 = $num1;
        $this->num2 = $num2;
    }
    
    public function limitclass2() {
        echo "The numbers are ($this->num1) and ($this->num2)";
    }
}

class Add extends Limit {
    public function addclass() {
        $a = ($this->num1 + $this->num2);
        echo "$this->num1 + $this->num2 = $a";
    }
}

class Minus extends Add {
    public function minusclass() {
        $b = ($this->num1 - $this->num2);
        echo "<br>$this->num1 - $this->num2 = $b";
    }
}

class Div extends Minus {
    public function divclass() {
        $c = ($this->num1 / $this->num2);
        echo "<br>$this->num1 / $this->num2 = $c";
    }
}

$answer = new Div(20, 5);
$answer->addclass();
$answer->minusclass();
$answer->divclass();
?>

Hierarchical Inheritance

<?php
class Animal {
    public function sleep() {
        echo "Sleeping<br>";
    }
}

class Dog extends Animal {
    public function bark() {
        echo "Woof!<br>";
    }
}

class Cat extends Animal {
    public function meow() {
        echo "Meow!<br>";
    }
}

$dog = new Dog();
$dog->sleep();
$dog->bark();

$cat = new Cat();
$cat->sleep();
$cat->meow();
?>

Multiple Inheritance (via Interfaces)

<?php
interface A {
    public function methodA();
}

interface B {
    public function methodB();
}

class C implements A, B {
    public function methodA() {
        echo "Method A from Interface A<br>";
    }
    
    public function methodB() {
        echo "Method B from Interface B<br>";
    }
}

$obj = new C();
$obj->methodA();
$obj->methodB();
?>

Hybrid Inheritance

PHP doesn't support true hybrid inheritance directly, but you can simulate it using classes and interfaces combined.

<?php
interface CanFly {
    public function fly();
}

interface CanSwim {
    public function swim();
}

trait AnimalSound {
    public function makeSound() {
        echo "Animal makes a sound<br>";
    }
}

class Animal1 {
    public function eat() {
        echo "Animal is eating<br>";
    }
}

class Bird extends Animal1 implements CanFly, CanSwim {
    use AnimalSound;
    
    public function fly() {
        echo "Bird is flying<br>";
    }
    
    public function swim() {
        echo "Bird is swimming<br>";
    }
}

class Duck extends Bird {
    public function quack() {
        echo "Duck says quack!<br>";
    }
}

$duck = new Duck();
$duck->eat();        // From Animal
$duck->fly();        // From CanFly interface
$duck->swim();       // From CanSwim interface
$duck->makeSound();  // From trait
$duck->quack();      // From Duck itself
?>

5. Polymorphism

Polymorphism allows objects of different classes to be treated as objects of a common super class. 

<?php
interface Shape {
    public function calculateArea();
}

class Circle implements Shape {
    private $radius;
    
    public function __construct($radius) {
        $this->radius = $radius;
    }
    
    public function calculateArea() {
        return pi() * pow($this->radius, 2);
    }
}

class Rectangle implements Shape {
    private $width;
    private $height;
    
    public function __construct($width, $height) {
        $this->width = $width;
        $this->height = $height;
    }
    
    public function calculateArea() {
        return $this->width * $this->height;
    }
}

$circle = new Circle(5);
$rectangle = new Rectangle(4, 6);

$shapes = [$circle, $rectangle];

foreach ($shapes as $shape) {
    echo "Area: " . $shape->calculateArea() . "<br>";
}
?>

Explanation: Both Circle and Rectangle implement the Shape interface, allowing them to be treated uniformly.

6. Encapsulation

Encapsulation is the bundling of data with the methods that operate on that data, often by restricting direct access to some of the object's components. 

<?php
class BankAccount {
    private $balance = 0;
    
    public function deposit($amount) {
        if ($amount > 0) {
            $this->balance += $amount;
            return true;
        }
        return false;
    }
    
    public function withdraw($amount) {
        if ($amount > 0 && $amount <= $this->balance) {
            $this->balance -= $amount;
            return true;
        }
        return false;
    }
    
    public function getBalance() {
        return $this->balance;
    }
}

$account = new BankAccount();
$account->deposit(1000);
$account->withdraw(500);
echo "Current balance: " . $account->getBalance();
?>

Explanation: The balance property is private and can only be modified through controlled methods.

7. Abstraction

Abstraction means hiding the complex implementation details and showing only the necessary features of an object. 

<?php
abstract class Vehicle {
    abstract public function startEngine();
    abstract public function stopEngine();
    
    public function honk() {
        echo "Honk! Honk!<br>";
    }
}

class Car extends Vehicle {
    public function startEngine() {
        echo "Car engine started<br>";
    }
    
    public function stopEngine() {
        echo "Car engine stopped<br>";
    }
}

class Motorcycle extends Vehicle {
    public function startEngine() {
        echo "Motorcycle engine started<br>";
    }
    
    public function stopEngine() {
        echo "Motorcycle engine stopped<br>";
    }
}

$car = new Car();
$car->startEngine();
$car->honk();
$car->stopEngine();

$bike = new Motorcycle();
$bike->startEngine();
$bike->stopEngine();
?>

Explanation: The abstract Vehicle class defines what methods must be implemented, but not how they should be implemented.
